あなたは、RAG用Markdown化の専門担当です。
以下の文書を、親子チャンク型RAGに適したMarkdownへ変換してください。

# 目的

- 後工程で見出し単位の親チャンク、検索用の子チャンクを作成します。
- 親チャンクで回答生成するため、章・節・条・項の構造を崩さないでください。
- 子チャンク検索のため、見出しだけで意味が通じるようにしてください。

# 変換ルール

1. 見出し構造
   - 文書タイトル: `#`
   - 章: `#` または `##`
   - 節: `##`
   - 条: `###`
   - 項・号: Markdownの番号付きリストまたは本文として保持
   - 原文の階層が曖昧な場合、条文単位を優先して見出し化する

2. 原文保持
   - 条文・定義・条件・例外規定は省略しない
   - 表はMarkdown表にする
   - 表が複雑な場合は、表の意味が検索できるようにテキストでも補足する
   - 注記、ただし書き、括弧書きは削らない

3. 検索性
   - 略称だけでなく、正式名称がある場合は正式名称を残す
   - 様式名は正式名称のまま残す
   - 「前条」「次項」などの参照は、可能なら見出し名を補足する
   - PDF由来のページ番号、ヘッダー、フッター、改行崩れは除去する

4. YAMLメタデータ
   各Markdownファイルの先頭に以下を付ける。

```yaml
---
title: 文書タイトル
corpus_id: 任意のcorpus_id
source: 元ファイル名
---
```

5. 禁止事項
   - 原文にない解釈を追加しない
   - 制度説明を勝手に補足しない
   - 要約だけにしない
   - 条番号を欠落させない

# 出力形式

Markdown本文のみ出力してください。
