# 文書更新マニュアル

## 文書の追加

新しい業務文書を追加する場合は、`data/markdown/<corpus_id>/` 配下にMarkdownファイルを配置します。
Markdownの先頭には、タイトル、文書ID、文書種別、版日などのYAMLメタデータを記載します。

## インデックス更新

文書を追加または更新した後は、チャンク作成、SearchTag作成、インデックス作成の順にスクリプトを実行します。
インデックス作成後は、ユーザーアプリで代表的な質問を投げて根拠が表示されることを確認します。

## 承認済みQA

よく使う質問と安定した回答は、管理者アプリから承認済みQAとして登録できます。
承認済みQAには別名質問を追加でき、ユーザーの表現揺れに対応できます。
