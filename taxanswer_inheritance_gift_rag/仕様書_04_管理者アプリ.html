仕様書 04: 管理者アプリ

管理者アプリはFastHTMLのRAG改善コンソールです。実装本体は app_fasthtml_admin_common/core.py、起動口は app_fasthtml_admin_common/main.py です。

起動

python -m app_fasthtml_admin_common.main

host/portは config/settings.json の urls.admin_bind_host / urls.admin_bind_port を使います。RAG API base URLは urls.rag_api_base_url から取得し、画面内で持ち回りません。

6画面構成

画面目的
ダッシュボード質問数、根拠なし率、低信頼率、未対応報告、QAヒット率、Index警告を確認。
改善キュー報告、根拠なし、低信頼、惜しいキャッシュミス、alias未生成QAなどを優先度順に表示。
報告対応ユーザー報告、回答時根拠、原因分類、対応方法、管理メモを扱う。
承認QAQA一覧、作成、編集、無効化、根拠検索、類似QA、alias管理、match-debug。
ログ探索質問ログ、回答元、cache miss理由、候補QA、根拠数、最大スコアを探索。
文書/評価chunk/index/SearchTag/chunk_report/eval状態を確認。

承認QA編集

  • QA一覧は1カラムで広く表示する。
  • 詳細・編集は大きめモーダルで表示する。
  • モーダル内で質問、回答、根拠JSON、version、memo、alias一覧、alias追加/更新/無効化、LLM alias生成を扱う。
  • disabled aliasは行全体をグレーアウトし、有効/無効が一目で分かるようにする。
  • LLM生成aliasは基本的に保存し、不要なものは管理者が無効化する。競合aliasは安全側でdisabled保存する。

SearchTagの扱い

  • SearchTagは補助機能として残す。
  • 管理画面の主導線では、QA alias追加、承認QA化、文書不足確認を優先する。
  • 表示や候補出しは search_tags.admin_edit_enabled と search_tags.show_improvement_candidates で制御する。

認証

現行スコープでは認証・認可は実装しません。ネットワークや端末側の運用制御で扱います。
