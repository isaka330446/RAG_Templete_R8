仕様書 05: RAG API

FastAPIで提供するユーザー向けAPI、管理API、評価・状態確認APIの最新版です。

起動と設定

python scripts/run_api.py
  • bind host/portは config/settings.json の urls.api_bind_host / urls.api_bind_port。
  • CORSは urls.cors_allow_origins。
  • LLM、Embedding、Reranker、alias/eval LLM接続先は urls セクションから取得する。

ユーザー向けAPI

Endpoint用途主なレスポンス
POST /ask質問に対するRAG回答を返す。answer, sources, debug, answer_source, cache_hit, qa_cache_id, cache_similarity
POST /ask_events工程イベント付きで質問処理を行う。progress events, final answer
POST /reports回答へのユーザー報告を登録。report_id, status
GET /health軽量ヘルスチェック。status
GET /health/deepログDB、QAキャッシュ、retrieverなどの状態確認。logging, answer_cache, retriever

回答処理のdebug情報

cache hit/missの理由を管理・改善に使うため、debugには次の情報を含めます。

  • cache_lookup_query と retrieval_query
  • matched_alias_id, matched_alias_text, matched_alias_type
  • cache_match_method, cache_miss_reason
  • cache_candidate_qa_id, cache_candidate_alias_id, cache_candidate_similarity
  • answer_source, qa_cache_id, cache_similarity

管理API

Endpoint用途
GET /admin/ops/summaryダッシュボードKPI、品質状態、Index状態。
GET /admin/actions/improvement-candidates改善キュー。no_hit、low_confidence、near_cache_miss、qa_alias_missingなど。
GET /admin/reports, GET /admin/reports/{id}報告一覧・詳細。
POST /admin/reports/{id}/status報告ステータス更新。
GET /admin/logs/statusログSQLite状態。
GET /admin/logs/searchログ探索。
GET /admin/index/statuschunk/index/SearchTag/chunk_report状態。
GET /admin/eval/status評価セットと評価結果状態。

承認QA / alias API

Endpoint用途
GET /admin/qa-cache承認QA一覧。
POST /admin/qa-cache承認QA新規登録。必要に応じてalias生成。
PATCH /admin/qa-cache/{qa_id}承認QA更新。質問/回答/根拠変更時はLLM aliasを要確認化。
POST /admin/qa-cache/{qa_id}/disable承認QA無効化。
GET /admin/qa-cache/{qa_id}/aliasesalias一覧。
POST /admin/qa-cache/{qa_id}/aliases手動alias追加。
PATCH /admin/qa-cache/aliases/{alias_id}alias編集、有効/無効切替。
POST /admin/qa-cache/{qa_id}/aliases/generateLLM alias生成。dry-run/保存/置換に対応。
POST /admin/qa-cache/backfill-aliases既存QAへのoriginal/LLM alias補完。
POST /admin/qa-cache/reload-indexalias in-memory index reload。
POST /admin/qa-cache/match-debug質問がなぜhit/missするかを確認。
GET /admin/qa-cache/alias-conflicts同一normalized aliasが複数QAに存在する競合を確認。

互換性

既存レスポンス項目は可能な限り維持し、新しい管理・debug項目は追加項目として扱います。
