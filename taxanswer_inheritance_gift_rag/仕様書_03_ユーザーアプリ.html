仕様書 03: ユーザーアプリ

一般ユーザー向けチャットアプリは app_fasthtml_modern/main.py のみです。

起動

python -m app_fasthtml_modern.main

host/portは config/settings.json の urls.frontend_bind_host / urls.frontend_bind_port を使います。RAG API接続先は urls.rag_api_base_url から取得します。

画面構成

  • 左端に細いレールを固定し、アイコンを上揃えで表示する。
  • 通常時はレールのみ、ホバー/フォーカスでラベル付きメニューを展開する。
  • 初期画面は中央ヒーロー、質問入力フォーム、質問例カードを表示する。
  • 質問送信後も入力フォームのデザインは初期画面と統一し、回答部分を不要な白枠で囲わない。
  • 新規チャットはChat Add Onアイコンとして、画面状態に応じて自然に配置する。

初期画面

  • 見出し: 「今日は何を確認しますか？」
  • 説明: 「タックスアンサー、通達、質疑応答事例を横断的に、相続税・贈与税・財産評価の判断材料を確認できます。」
  • 入力欄は見出しと説明文の直下、中央軸に揃える。
  • 右側または下部に質問例カードを表示し、クリックで入力欄へ反映する。
  • 質問例は承認済みQAの新しいものを優先し、取得できない場合は静的fallbackを使う。

回答表示

  • RAG回答は全幅を使って読みやすく表示する。
  • 回答下部に根拠チップ、報告ボタン、承認済みQA登録ボタンを表示する。
  • 根拠なし/低信頼時はユーザー向けの次アクションを表示する。
  • 回答生成中は工程表示を行い、現在工程の点だけをアニメーションさせる。

根拠表示

  • 根拠表示は大型ポップアップ方式のみ。
  • side drawer方式とUIモード切替は廃止。
  • 本文中の「根拠1」「根拠①」「[2]」「根拠: タイトル」などを可能な範囲で根拠リンク化する。
  • ポップアップは背景オーバーレイ、閉じるボタン、ESC、背景クリックで閉じる。
  • 根拠タイトル、文書種別、出典ファイル、見出し、本文preview/親chunkを表示する。
