仕様書 02: DB

SQLite DB、主要テーブル、起動時チェック、運用上の注意です。

SQLite運用方針

  • SQLiteは本番必須機能として扱う。
  • 標準は journal_mode=DELETE。これはDB削除ではなくrollback journal方式。
  • logging.required=true、answer_cache.required=true が標準。
  • required=true で初期化、schema作成、起動時書き込みチェックに失敗した場合、APIは起動しない。
  • 開発用途でのみ required=false によりdisabled継続を許可できる。

起動時チェック

mkdir(parent)
sqlite3.connect(timeout=settings)
PRAGMA busy_timeout
PRAGMA foreign_keys=ON
PRAGMA journal_mode=DELETE
CREATE TABLE IF NOT EXISTS ...
startup write check INSERT/DELETE/COMMIT

失敗時はDBパス、journal_mode、例外内容を含むRuntimeErrorで停止します。
